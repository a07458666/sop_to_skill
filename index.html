SOP-to-Skill Compiler

將繁複的「標準作業程序（SOP）」文檔，編譯為可被 Agent 載入的合法 Skill 目錄。

設計思維：為什麼要轉換？

傳統 Markdown SOP

  • 注意力渙散：AI 閱讀長文容易跳步或忽視規則。
  • 安全盲點：無法硬性禁止 AI 跳過特定核准步驟。
  • 高 Token 消耗：每次對話都得重傳整份文件。
  • 難以除錯：出錯時不知卡在流程的哪個分支。

合法 Skill

  • 標準入口：生成必要的 SKILL.md 與 YAML frontmatter。
  • 硬性安全阻斷：可在特定狀態加上人機協同審核閘。
  • Bundled Resource：以 flow.json 保存狀態機。
  • 清晰狀態鏈：完美的狀態遷移日誌（State History）。
