SOP-to-Skill Compiler

將繁複的「標準作業程序（SOP）」文檔，編譯為可被 Agent 載入的合法 Skill 目錄。

設計思維：為什麼要轉換？

傳統 Markdown SOP

  • 注意力渙散：AI 閱讀長文容易跳步或忽視規則。
  • 安全盲點：無法硬性禁止 AI 跳過特定核准步驟。
  • 高 Token 消耗：每次對話都得重傳整份文件。
  • 難以除錯：出錯時不知卡在流程的哪個分支。

合法 Skill

  • 標準入口：生成必要的 SKILL.md 與 YAML frontmatter。
  • 硬性安全阻斷：可在特定狀態加上人機協同審核閘。
  • Bundled Resource：以 flow.json 保存狀態機。
  • 清晰狀態鏈：完美的狀態遷移日誌（State History）。

MCP Server 掛載與工具驗證 (Runtime Mount & Validation)

編譯時會驗證每個 API / MCP 工具的「參數契約」與「回傳判讀規則」（詳見上方 SOP 品質報告）。MCP 工具需先掛載對應 server，下方執行模擬器才能成功呼叫；API 工具則以 REST endpoint 直接呼叫。

Skill 執行模擬器 (API / MCP Call Simulation)

從 flow.json 的 start state 開始。每個工具 state 會模擬實際的 API / MCP 呼叫與回傳，並示範 Agent 如何判讀回傳值以決定下一步路由。
